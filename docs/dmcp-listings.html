Overview

This document records the addition of two supplemental data sources to the ICE detention pipeline: point-in-time DMCP (Detention Management Compliance Program) facility authorization rosters from December 2015 and July 2017.

These are a different document type from the annual FY stats spreadsheets (FY2019–FY2026) that form the main panel. Rather than single-year snapshots, each is an authorization roster showing all currently authorized ICE detention facilities at a point in time, with historical ADP data spanning multiple fiscal years per row, alongside contract, operator, and inspection metadata.

The two lists are tracked as pipeline targets faclist15 and faclist17 (see _targets.R). Full provenance metadata is in data/dmcp-listings.yml.

Data sources

faclist15 — December 2015 XLSX

  • Title: 2015 ICE Detention Facility Listing
  • Source: ICE FOIA reading room (ice.gov/foia)
  • URL: https://www.ice.gov/doclib/foia/dfs/2015IceDetentionFacilityListing.xlsx
  • File: data/2015IceDetentionFacilityListing.xlsx
  • Data as of: December 8, 2015
  • Currency note: FY2016 data reflects IIDS as of 12/08/2015 (approximately the first 70 days of FY2016); FY2015 and earlier reflect end-of-fiscal-year figures.
  • Fiscal years covered: FY2010–FY2016 (ADP columns fy10_adp–fy16_adp)

faclist17 — July 2017 PDF

  • Title: ICE DMCP Facility List — ERO Custody Management Division, FY2017
  • Source: Prison Legal News document library
  • URL: https://www.prisonlegalnews.org/media/publications/ICE_DMCP_Facility_List_ERO_Custody_Management_Division_2017.pdf
  • File: data/ICE_DMCP_Facility_List_2017.pdf
  • Data as of: July 10, 2017
  • Currency note: ADP figures reflect IIDS as of 07/10/2017; EID data through 07/08/2017.
  • Fiscal years covered: FY2010–FY2017 (ADP columns fy10_adp–fy17_adp)
  • Parsing: Extracted with tabulapdf::extract_tables() using explicit column boundaries derived from bounding-box coordinates of the PDF header row.

Note on two unmatched adjacent-facility DETLOCs

Two facilities — JAMESGA (Folkston IPC / D. Ray James, GA) and OTROPNM (Otero County Prison, NM) — share a city and road with canonical FY19–26 facilities but have distinct street addresses, indicating separate buildings within the same detention complex. These are kept as distinct entries in the canonical list pending confirmation of whether the companion buildings have their own separate operational roles in the current DDP data. (A third facility, STCDFTX / South Texas Detention Complex, was originally in this group but was subsequently confirmed as the same facility as canonical 343 / South Texas ICE Processing Center at 566 Veterans Drive, Pearsall TX.)

The Deportation Data Project (DDP) provides daily detention facility population data spanning September 2023 to October 2025. This represents real-time operational detention, and includes 853 distinct facility codes. Of the 174 canonical FY2019–FY2026 facilities that do NOT appear in DMCP data (2015–2017), 135 have representation in the DDP data.

Data source

  • Title: Deportation Data Project daily population data
  • Date range: 2023-09-01 to 2025-10-15 (776 days)
  • Distinct facilities: 853 facility codes across all states + territories
  • Variables: detention_facility_code, detention_facility (name), state, date, population counts by gender/criminal status/age

The DDP data was imported as a feather file and cleaned using the same facility name standardization rules applied to DMCP data (abbreviation expansion, acronym restoration, truncation correction).

Building DDP match tables

The matching logic lives in build_ddp_canonical_map() (in R/ddp.R) and runs as a pipeline target. Manual strong matches are defined in ddp_manual_strong_matches(). Below we derive display tables from the pipeline target ddp_canonical_map.

Code 
# Derive display tables from the pipeline target.
# Join canonical name/city for display.
ddp_display <- ddp_canonical_map |>
  left_join(
    canonical_no_dmcp |> select(canonical_id, canonical_name, facility_city,
                                 facility_state),
    by = "canonical_id"
  )

ddp_exact_matches <- ddp_display |>
  filter(match_type == "exact") |>
  rename(can_state = facility_state, ddp_code = detloc) |>
  arrange(can_state, canonical_name)

ddp_county_matches <- ddp_display |>
  filter(match_type == "partial_county") |>
  rename(can_state = facility_state, ddp_code = detloc) |>
  arrange(can_state, canonical_name)

ddp_keyword_matches <- ddp_display |>
  filter(match_type == "partial_keyword") |>
  rename(can_state = facility_state, ddp_code = detloc) |>
  arrange(can_state, canonical_name)

# Fully unmatched: not in DMCP and not in DDP
all_matched_ids <- unique(ddp_canonical_map$canonical_id)

ddp_fully_unmatched <- canonical_no_dmcp |>
  filter(!(canonical_id %in% all_matched_ids)) |>
  left_join(
    facility_presence |> select(canonical_id, FY19:FY26),
    by = "canonical_id"
  ) |>
  select(canonical_id, canonical_name, facility_city, facility_state,
         FY19:FY26) |>
  arrange(facility_state, canonical_name)
